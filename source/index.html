Academy API GitHub

Добро пожаловать на страницу тестирования Academy API.
Данная библиотека была написана в качестве решения одной из задач поставленных компанией Яндекс.

Список методов

  • Speaker – класс, отвечающий за работу с преподавателя. Включает методы:
    • add(name, company, photo) – метод добавляет нового преподавателя;
      • name – имя преподавателя;
      • company – компания, в которой работает преподаватель;
      • photo – ссылка на фотографию преподавателя.
    • edit(name, arg, value) – метод изменяет имя заданного преподавателя;
      • name – имя преподавателя;
      • arg – аргумент, который требуется изменить;
      • value – новое значение.
    • remove(name) – метод удаляет заданного преподавателя;
      • name – имя преподавателя.
    • search(name, time) – метод осуществляет поиск лекций в заданном интервале, которые проводил преподаватель;
      • name – имя преподавателя;
      • time – временной интервал (например '01.04.2017 10:00, 01.04.2017 20:00').
    • _get(name) – метод возвращает заданного преподавателя, либо всех существующих;
      • name – имя преподавателя (не обязательный аргумент).
  • Lecture – класс, отвечающий за работу с лекциями.
    • add(name, speaker, auditorium, time, schools, link) – метод добавляет новую лекцию;
      • name – название лекции;
      • speaker – имя преподавателя, читающего лекцию;
      • auditorium – аудитория, в которой будет проведена лекция;
      • time – временной интервал (например '01.04.2017 10:00, 01.04.2017 20:00');
      • schools – школы, для которых предназначена лекция;
      • link – ссылка на материалы.
    • edit(name, arg, value) – метод изменяет значение лекции;
      • name – название лекции;
      • arg – аргумент, который требуется изменить;
      • value – новое значение.
    • remove(name) – метод удаляет заданную лекцию;
      • name – название лекции.
    • search(time) – метод осуществляет поиск лекций в заданном интервале;
      • time – временной интервал (например '01.04.2017 10:00, 01.04.2017 20:00').
    • _get(name) – метод возвращает заданную лекцию, либо все существующие;
      • name – название лекции (не обязательный аргумент).
  • Auditorium – класс, отвечающий за работу с аудиториями.
    • add(name, capacity, location) – метод добавляет новую аудиторию;
      • name – название аудитории;
      • capacity – вместимость аудитории;
      • location – местоположение.
    • edit(name, arg, value) – метод изменяет значение аудитории на новое.
      • name – название аудитории;
      • arg – аргумент, значение которого надо изменить;
      • value – новое значение.
    • remove(name) – метод удаляет заданную аудиторию;
      • name – название аудитории.
    • search(name, time) – метод осуществляет поиск лекций в заданном интервале, в которой аудитории она проводилась;
      • name – название аудитории;
      • time – временной интервал (например '01.04.2017 10:00, 01.04.2017 20:00').
    • _get(name) – метод возвращает заданную аудиторию, либо все существующие;
      • name – название аудитории (не обязательный аргумент).
  • School – класс, отвечающий за работу со школами.
    • add(name, students) – метод добавляет новую школу;
      • name – название школы;
      • students – количество студентов.
    • edit(name, arg, value) – метод изменяет значение школы на новое;
      • name – название школы;
      • arg – аргумент, который требуется изменить;
      • value – новое значение.
    • remove(name) – метод удаляет заданную школу;
      • name – название школы.
    • search(name, time) – метод осуществляет поиск лекций в заданном интервале, в которых участвовала заданная школа;
      • name – название школы;
      • time – временной интервал (например '01.04.2017 10:00, 01.04.2017 20:00').
    • _get(name) – метод возвращает заданную школу, либо все существущие;
      • name – название школы (не обязательный аргумент).
  • TestAPI() – функция для тестирования библиотека (очень примитивная). Принимает значения:
    • teacher или 1 – тестирование класса Teacher;
    • lecture или 2 – тестирование класса Lecture;
    • school или 3 – тестирование класса School;
    • auditorium или 4 – тестирование класса Auditorium;

Использование

Чтобы начать работу с данным API, в ознакомительных целях, откройте консоль вашего браузера.
Вам доступны следующие команды и возможности (примеры):

  • teacher.search('Яковлев', '01.04.2017 10:00, 01.04.2017 16:00')
  • lecture.edit('Организация мероприятий', 'name', 'Управление персоналом')
  • auditorium.get('Вертикаль')
  • school.getAll()
  • TestAPI('lecture') – примитивный тест для класса Lectures.
